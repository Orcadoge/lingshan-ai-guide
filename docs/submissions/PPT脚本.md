**正文**：
> "智游灵山"是一款面向灵山胜境与拈花湾景区的AI数字人智能导游系统。我们以多模态大模型为核心，融合语音对话、数字人交互、情感分析、个性化路线推荐与数据运营分析，为游客提供"听得懂、看得见、有温度"的智慧导览体验。

## 第3页 · 核心痛点（45秒）

**标题**：传统景区导览的三大痛点

| 痛点 | 具体表现 | 游客感受 |
|------|---------|---------|
| 信息碎片化 | 纸质地图、指示牌、人工讲解各自为政 | "看完就忘，不知道该往哪走" |
| 服务同质化 | 千篇一律的讲解词，无法匹配个人兴趣 | "我对佛教文化感兴趣，导游却在讲建筑结构" |
| 反馈无闭环 | 游客满意度无法量化，运营靠经验 | "不知道游客到底喜不喜欢这个景点" |

**底部金句**：
> "游客需要的不是更多信息，而是更懂自己的服务。"

## 第10页 · 系统架构（30秒）

**标题**：云端+本地协同的弹性架构

**架构图**：（简化版，突出核心组件）

```
游客端(Next.js) ──→ NestJS API ──→ 火山方舟LLM
    │                    │               │
    │                    ├─→ ChromaDB   └──→ 情感分析
    │                    │    (RAG)
    │                    └─→ PostgreSQL + Redis
    │                         (数据持久化)
    │
    └─→ 浏览器TTS / WebSocket音频
```

**核心亮点**：
- Turborepo Monorepo：前后端统一仓库，代码复用率高
- Zod全链路校验：API请求→数据库写入，类型安全零漏洞
- Prisma统一ORM：模型即类型，自动生成迁移脚本
