## 一、环境要求

### 1.1 硬件配置

| 环境 | CPU | 内存 | 硬盘 | 说明 |
|------|-----|------|------|------|
| 本地开发 | 4核+ | 8GB+ | 20GB+ | 可运行全量服务 |
| 演示部署 | 2核+ | 4GB+ | 10GB+ | 基础功能演示 |
| 生产环境 | 4核+ | 16GB+ | 50GB+ | 并发用户>100 |

### 1.2 软件依赖

| 软件 | 版本 | 用途 |
|------|------|------|
| Node.js | 20.x LTS | 运行前后端 |
| pnpm | 9.x+ | 包管理（Monorepo必需） |
| Docker | 24.x+ | 数据库容器化 |
| Docker Compose | 2.x+ | 多服务编排 |
| Git | 2.x+ | 代码管理 |

### 1.3 外部服务（可选）

| 服务 | 用途 | 是否必需 |
|------|------|---------|

## 二、快速启动（本地开发）

### 2.1 克隆代码

```bash
git clone https://github.com/Orcadoge/lingshan-ai-guide.git
cd lingshan-ai-guide
```

### 2.2 安装依赖

```bash
# 安装 pnpm（如果尚未安装）
npm install -g pnpm

# 安装项目依赖
pnpm install
```

### 2.3 配置环境变量

```bash
# 复制示例配置文件
cp .env.example .env

# 编辑 .env，填入你的配置
```

**必需配置项**：

```env
# 数据库
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/lingshan_ai_guide?schema=public"

# Redis
REDIS_URL="redis://localhost:6379"

# 火山方舟（可选，无Key时降级为规则兜底）
ARK_API_KEY="your_ark_api_key"
ARK_MODEL="ep-xxxxxxxxxxxxx"  # 你的模型接入点ID

# 豆包实时语音（可选）
VOLCENGINE_APP_ID="your_app_id"
VOLCENGINE_ACCESS_TOKEN="your_access_token"
VOLCENGINE_USER_ID="your_user_id"
```

### 2.4 启动数据库依赖

```bash
# 使用 Docker Compose 启动 PostgreSQL + Redis + ChromaDB
docker-compose up -d postgres redis chromadb

# 等待数据库就绪（约10秒）
docker-compose ps
```

### 2.5 数据库迁移

```bash
cd packages/data

# 执行 Prisma 迁移（创建所有表）
npx prisma migrate dev --name init

# 生成 Prisma Client 类型
npx prisma generate

# 导入种子数据（灵山景点数据）
npx prisma db seed
```

### 2.6 导入 RAG 知识库

```bash
# 进入后端目录
cd apps/api

# 安装 Python 依赖（用于数据导入）
pip install pandas openpyxl chromadb

# 执行数据导入脚本
python scripts/ingest_rag_data.py --source ./data/lingshan_data.xlsx
```

### 2.7 导入游客行为数据（可选）

```bash
# 14万游客行为数据导入（用于数据大屏统计）
cd packages/data
node scripts/import-visitor-behavior.js ./visitor-behavior-data.xlsx
```

### 2.8 启动开发服务器

```bash
# 在项目根目录启动所有服务
cd ../../  # 回到项目根目录
pnpm dev

# 或单独启动前端/后端
pnpm dev --filter=web    # 仅前端
pnpm dev --filter=api    # 仅后端
```

**访问地址**：
- 前端：`http://localhost:3000`
- 后端 API：`http://localhost:3001`
- 后端 Swagger：`http://localhost:3001/api-docs`

## 三、Docker 生产部署

### 3.1 一键部署

```bash
# 项目根目录下执行
docker-compose -f docker-compose.prod.yml up -d
```

### 3.2 生产 Docker Compose 配置

```yaml
# docker-compose.prod.yml
version: '3.8'

services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./apps/web/public:/usr/share/nginx/html/static
    depends_on:
      - web
      - api
    restart: unless-stopped

  web:
    build:
      context: .
      dockerfile: ./apps/web/Dockerfile
    environment:
      - NEXT_PUBLIC_API_URL=http://api:3001
    depends_on:
      - api
    restart: unless-stopped

  api:
    build:
      context: .
      dockerfile: ./apps/api/Dockerfile
    environment:
      - DATABASE_URL=postgresql://postgres:postgres@postgres:5432/lingshan_ai_guide?schema=public
      - REDIS_URL=redis://redis:6379
      - ARK_API_KEY=${ARK_API_KEY}
      - ARK_MODEL=${ARK_MODEL}
    depends_on:
      - postgres
      - redis
      - chromadb
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    environment:
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
      - POSTGRES_DB=lingshan_ai_guide
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    restart: unless-stopped

  chromadb:
    image: chromadb/chroma:0.5.0
    volumes:
      - chroma_data:/chroma/chroma
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
  chroma_data:
```

### 3.3 Nginx 反向代理配置

```nginx
# nginx.conf
events {
    worker_connections 1024;
}

http {
    upstream web {
        server web:3000;
    }
    upstream api {
        server api:3001;
    }

    server {
        listen 80;
        server_name localhost;

        # 前端静态资源
        location / {
            proxy_pass http://web;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
        }

        # API 代理
        location /api/ {
            proxy_pass http://api/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # WebSocket 代理（实时语音）
        location /voice/realtime {
            proxy_pass http://api;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }
    }
}
```

## 五、常见问题

### 5.1 安装问题

**Q: `pnpm install` 报错，提示无法解析 workspace 依赖**

```bash
# 解决方案：确保在 Monorepo 根目录执行，且 pnpm-workspace.yaml 存在
cat pnpm-workspace.yaml
# 应输出：packages: ['apps/*', 'packages/*']

# 如果缺失，重新创建
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

**Q: Prisma 迁移报错 `P1001: Can't reach database server`**

```bash
# 检查数据库是否启动
docker-compose ps

# 如果 postgres 未启动
docker-compose up -d postgres

# 等待 5 秒后重试
npx prisma migrate dev
```

### 5.2 运行时问题

**Q: 前端报 `API connection refused` 错误**

```bash
# 检查后端是否启动
curl http://localhost:3001/health

# 检查前端代理配置
# apps/web/next.config.ts 中应有：
# async rewrites() { return [{ source: '/api/:path*', destination: 'http://localhost:3001/:path*' }] }
```

**Q: TTS 无声音，浏览器控制台报错**

```bash
# 检查浏览器是否支持 speechSynthesis
# Chrome/Edge 支持最佳，Safari 需用户交互触发

# 检查是否有火山方舟 Key（无 Key 时用浏览器 TTS 兜底）
# 确保浏览器未静音，且允许网站播放音频
```

**Q: RAG 检索返回空结果**

```bash
# 检查 ChromaDB 是否启动
docker-compose ps chromadb

# 检查知识库是否已导入
curl http://localhost:8000/api/v1/collections/lingshan_knowledge/count

# 重新导入数据
python scripts/ingest_rag_data.py --source ./data/lingshan_data.xlsx --force
```

### 5.3 部署问题

**Q: Docker 构建失败，提示 `Cannot find module '@handan/shared'`**

```bash
# 解决方案：确保 Docker build 在 Monorepo 根目录执行
# Dockerfile 中应有：
# COPY pnpm-workspace.yaml turbo.json ./
# COPY packages ./packages
# COPY apps ./apps
```

**Q: 生产环境数据库连接超时**

```bash
# 增加 Prisma 连接池配置
# packages/data/prisma/schema.prisma 中：
# datasource db {
#   provider = "postgresql"
#   url      = env("DATABASE_URL")
#   directUrl = env("DATABASE_URL")  # 用于迁移
# }
```
