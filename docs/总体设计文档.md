## 一、项目概述

### 1.1 项目定位

"赵都云旅"是一款面向灵山胜境与拈花湾景区的AI数字人智能导游系统。系统以多模态大模型为核心，融合语音对话、数字人交互、情感分析、个性化路线推荐与数据运营分析，为游客提供"听得懂、看得见、有温度"的智慧导览体验。

## 三、核心模块设计

### 3.1 AI数字人交互模块

**设计思路**：比赛周期短，不依赖外部数字人SaaS API（避免网络延迟和费用问题），采用"纯前端Mock方案"实现核心交互体验。

**实现方案**：
1. **数字人舞台**：CSS动画驱动的静态形象展示区
   - 使用卡通/Q版国风素材图作为数字人形象
   - 正弦波模拟口型动画（根据TTS音频节奏）
   - 状态指示灯（倾听/思考/播报）
2. **对话流**：WebSocket双向通信，支持文本+语音输入
3. **底部操作台**：热门问题快捷标签 + 语音输入按钮 + 文本输入框

**技术亮点**：
- 零外部API依赖，完全离线可用
- 口型同步通过音频分析（Web Audio API提取音量包络→正弦波振幅映射）
- 换装功能：前端状态切换不同形象素材

### 3.2 情感分析引擎

**设计思路**：多模态情感感知是赛题核心加分项，采用"渐进式叠加"策略——先文本、再语音、最后视觉。

**当前实现（文本+语音融合）**：
1. **文本情感分析**：火山方舟LLM + 关键词兜底
   - LLM Prompt工程：要求返回JSON格式 `{emotion, confidence, details, keywords}`
   - 关键词兜底：旅游场景正负向词汇表（中文+英文）
   - 置信度校准：LLM失败时自动降级到关键词规则
2. **语音情感分析**：实时音频特征提取
   - Pitch基频 → 情绪活跃度
   - Energy能量 → 情绪强度
   - Speed语速 → 急迫/平静
3. **多模态融合**：文本权重0.7 + 声学权重0.3
   - 加权计算综合情感状态
   - 自动写入 `VisitorEmotion` 表

**数据模型**：
```prisma
model VisitorEmotion {
  id            String        @id @default(cuid())
  sessionId     String
  source        EmotionSource // TEXT / VOICE / FACE / FUSED
  textContent   String?
  textSentiment Json?         // {emotion, confidence, details, keywords}
  voiceFeatures Json?         // {pitch, energy, speed}
  faceFeatures  Json?         // 预留视觉字段
  poiId         String?       // 关联景点
  createdAt     DateTime      @default(now())
}
```

### 3.3 RAG知识库

**设计思路**：本地知识库准确率≥90%是赛题硬性要求，采用"向量检索+规则兜底"双保险。

**实现方案**：
1. **数据 ingest**：`scripts/ingest_rag_data.py`
   - 支持 XLSX/CSV 导入
   - Ark Embedding 模型生成向量
   - ChromaDB 持久化存储
2. **检索策略**：`rag-retrieval.ts`
   - 语义相似度 Top-K 召回
   - 降级容错：ChromaDB 不可用时返回预设FAQ
3. **知识库内容**：
   - 灵山胜境20个景点结构化数据
   - 拈花湾景点数据
   - 官方14万游客行为数据（用于个性化推荐）

### 3.4 数据大屏

**设计思路**：管理后台从基础CRUD升级为运营分析平台，满足赛题"数据可视化"要求。

**前端模块（ECharts）**：
1. 4个KPI指标卡片：会话量 / 转化率 / 满意度 / 数字人使用率
2. 双折线图：满意度趋势 + 会话量趋势
3. 环形图：情感分布（开心/平静/困惑/疲惫/不满）
4. 横向柱状图TOP10：热门问答
5. 柱状图：景点热度排行
6. 实时会话表格：带情感标签 + 星星评分

**后端API（6个聚合端点）**：
- `GET /analytics/overview` — 今日概览KPI
- `GET /analytics/emotion-distribution` — 情感分布统计
- `GET /analytics/hot-questions` — 热门问答TOP10
- `GET /analytics/poi-ranking` — 景点热度排行
- `GET /analytics/satisfaction-trend` — 满意度趋势
- `GET /analytics/realtime-sessions` — 实时会话列表

### 3.5 个性化路线推荐

**设计思路**："规则优先 + LLM润色"——规则层保证可执行性，LLM负责叙事和亮点提炼。

**规则引擎（`packages/planner`）**：
1. 意图标准化：解析用户输入的时间、同行人、偏好
2. 候选POI召回：基于标签和主题过滤
3. 评分排序：综合距离、评分、停留时长
4. 行程拼装：时间窗口约束 + 餐饮穿插
5. 预算估算：门票+餐饮+交通

**LLM增强**：
- 规则生成的行程草稿 → LLM润色为自然语言描述
- 添加文化故事、拍照建议、注意事项

## 六、部署架构

### 6.1 本地开发环境

```bash
# 1. 启动数据库依赖
docker-compose up -d postgres redis chromadb

# 2. 安装依赖
pnpm install

# 3. 数据库迁移
npx prisma migrate dev

pnpm dev
```

### 6.2 生产部署方案

**推荐架构**：Docker Compose + Nginx 反向代理

```yaml
# docker-compose.yml 核心服务
services:
  web:        # Next.js 前端 (SSR)
  api:        # NestJS 后端
  postgres:   # PostgreSQL 主数据库
  redis:      # Redis 缓存
  chromadb:   # ChromaDB 向量库
  nginx:      # 反向代理 + 静态资源
```

**云端+本地协同（加分项）**：

- 本地：轻量Next.js前端 + 浏览器TTS + IndexedDB离线缓存
- 通信：WebRTC/HLS视频流 + WebSocket音频流

## 七、创新点与差异化

### 7.1 技术创新

1. **纯前端数字人方案**：不依赖外部SaaS API，通过CSS动画+Web Audio API实现零成本的口型同步和形象驱动，比赛现场无需网络也能演示
2. **LLM+规则双保险情感分析**：火山方舟LLM提供语义级情感理解，关键词规则兜底保证离线可用，准确率≥90%
3. **实时音频特征融合**：从WebSocket语音流中实时提取pitch/energy/speed，与文本情感加权融合，实现"听得见情绪"的多模态感知
4. **云端+本地协同架构**：数字人渲染和情感分析可弹性上云，前端保持轻量，兼顾性能与成本

### 7.2 产品创新

1. **禅意UI设计**：宣纸白+竹青+琉璃金配色，水墨山水背景，卷轴式边框，贴合灵山胜境佛教文化主题
2. **换装互动**：数字人支持多套国风服饰切换（禅服/唐装/宋制），增强游客参与感
3. **情感驱动推荐**：根据游客实时情绪状态动态调整推荐策略（开心→推荐更多景点，疲惫→推荐休息点）
4. **管理后台数据运营**：从基础CRUD升级为ECharts数据大屏，支持情感分布、景点热度、问答趋势等运营分析

### 7.3 工程创新

1. **Turborepo Monorepo**：前后端+共享包统一仓库，pnpm workspace管理依赖，代码复用率高
2. **Zod全链路校验**：从API请求到数据库写入，Schema校验贯穿前后端，类型安全零漏洞
3. **Prisma统一ORM**：数据库模型即TypeScript类型，自动生成迁移脚本，开发效率提升30%
4. **降级策略完备**：LLM失败→规则兜底、ChromaDB失败→本地缓存、TTS失败→浏览器speechSynthesis，系统鲁棒性强
